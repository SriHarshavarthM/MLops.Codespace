MLops.Codespace Project Status

The project now includes a working FastAPI backend, configuration-backed model and scenario endpoints, and an interactive dashboard for running security validation workflows.

Completed Work

  • Integrated the validation pipeline with the backend API so requests can trigger the model attack workflow end to end.
  • Added health, status, model catalog, and scenario catalog endpoints for the frontend and operators.
  • Replaced the placeholder UI with a React dashboard that surfaces backend status, validation controls, and metrics output.
  • Improved dataset detection to avoid misclassifying local CSV files as Kaggle datasets.
  • Added regression tests for health checks, model configuration, and the core attack-engine behavior.

How to Use It

  • Start the backend with uvicorn app.backend.main:app --host 0.0.0.0 --port 8000.
  • Serve the frontend from the Vite app in app/frontend once Node.js is available in the environment.
  • Use the dashboard to run a validation job, inspect drift results, and review the Prometheus metrics snapshot.

Current Focus

  1. Keep the backend and validation pipeline connected to the same data sources.
  2. Optionally wire the UI to a live model endpoint or external inference provider.
  3. Extend monitoring with richer dashboards and alerting once the deployment environment is available.
